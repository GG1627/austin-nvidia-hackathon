## How to run / test right now

```bash
# Full test suite (offline, no keys needed):
python3 -m unittest -v

# Interactive single cycle:
python3 main.py

# Scripted 4-run demo (what we'll show judges):
python3 main.py --reset
python3 main.py --cycles 4 --simulate

# Just the improvement dashboard:
python3 main.py --metrics

# Back to cold start:
python3 main.py --reset
```

With `NVIDIA_API_KEY` in `.env` (copy `.env.example`), recommendations are
generated by NVIDIA NIM (`meta/llama-3.1-70b-instruct`, override with
`NIM_MODEL`). Without a key — or if the call fails or returns output that
fails the quality validator — Agent 3 automatically uses its deterministic
engine, so the demo can never die on stage. Every cycle prints which engine
produced the recommendations.

## The interfaces Agent 3 calls (build to these)

Agent 3 takes its collaborators via constructor injection and only ever
calls these methods. Types are the dataclasses in `agents/contracts.py`
(mirroring docs/ARCHITECTURE.md) — please import them rather than
redefining.

### Agent 1 (memory) — required

```python
def get_creator_context(self) -> CreatorContext: ...

def ingest_feedback(self, feedback: Feedback,
                    recommendation: Recommendation | None = None) -> None: ...
```

The `recommendation` kwarg is optional: Agent 3 inspects your signature and
only passes it if you accept it (it carries `supporting_patterns` etc. so
you can update pattern confidence). A plain `ingest_feedback(feedback)`
also works.

### Agent 1 — optional (Agent 3 checks `hasattr` and degrades gracefully)

```python
def increment_run_count(self) -> int      # else Agent 3 counts its own log
def mark_surfaced(self, opportunity_id: str) -> None   # for Agent 2 dedup
def get_surfaced_topics(self) -> list[str]
def pattern_count(self) -> int
```

### Agent 2 (research) — required

```python
def get_opportunities(self, creator_context: CreatorContext) -> list[Opportunity]: ...
```

Optional: expose `last_duplicates_filtered: int` after each call — it feeds
the "duplicate research avoided" metric on the dashboard.

## The two-line swap

In `main.py`, `build_strategist()`:

```python
# before
memory = MockMemoryAgent()
research = MockResearchAgent(memory)

# after
memory = CreatorMemoryAgent(...)   # agents/agent1_memory.py
research = ResearchAgent(...)      # agents/agent2_research.py
```

Nothing else changes. You can also swap just ONE agent and keep the other
stub — they only talk to each other through the interfaces above.

To verify your real agent against Agent 3's expectations, point the tests'
factory at it (or just run `python3 main.py --simulate` after the swap):
`tests/test_agent3.py::make_strategist` is the single place stubs are wired.

## Files Agent 3 owns / writes

- `memory/cycle_history.json` — per-run log (recommendations, feedback,
  metrics). Owned by Agent 3; Agent 1 should not write it.
- `memory/knowledge_graph.json` — owned by Agent 1 (the stub writes it
  today using the exact schema from docs/ARCHITECTURE.md; the real Agent 1
  should keep that schema).

## Behavior guarantees Agent 3 enforces

- Every recommendation has: specific title, why-reasoning citing pattern
  ids + Agent 2 evidence, confidence 0–1, and ≥3 action steps (LLM output
  failing this is rejected and regenerated deterministically).
- Topics on the creator's avoid list and opportunities marked
  `already_surfaced` are never recommended.
- Ranks are recomputed locally by confidence — LLM ordering is not trusted.
- All feedback (accept / reject / defer + notes) is forwarded to Agent 1
  in the same cycle, before metrics are logged.
